## Why

As the Trend Trading System grows, a central, structured documentation repository is essential for onboarding, maintaining architectural consistency, and guiding future extensions. Currently, project knowledge is fragmented across implementation files and temporary changes.

## What Changes

- **Documentation Directory**: Creation of a dedicated `docs/` folder in the project root.
- **Project Overview**: Introduction of `docs/goals.md` and `docs/usage.md`.
- **Architectural Reference**: Creation of `docs/architecture.md` covering principles, modules, domain models, and structure.
- **Developer Guide**: Addition of `docs/extension-guide.md` to assist in adding new strategies and adapters.

## Capabilities

### New Capabilities
- `project-overview`: High-level goals and usage instructions for the trading system.
- `architectural-blueprint`: Detailed breakdown of the system's design, domain models, and engineering structure.
- `extension-framework`: Guidelines and patterns for extending the system with new modules.

### Modified Capabilities
- (None)

## Impact

This change adds static documentation files to the repository. It has no direct impact on the runtime behavior of the trading system but significantly improves the developer experience and maintainability.
